Deprecated
This feature is no longer recommended. Though some browsers might still support it, it may have already been removed from the relevant web standards, may be in the process of being dropped, or may only be kept for compatibility purposes. Avoid using it, and update existing code if possible; see the compatibility table at the bottom of this page to guide your decision. Be aware that this feature may cease to work at any time.

Sumari

<Isindex> és un element HTML obsolet que posa un camp de text en una pàgina per consultar el document. <Isindex> proporcionava una sola entrada de text de línia per ingressar una cadena de consulta. Quan s'envia, el servidor retorna una llista de pàgines que coincideixin amb la consulta. El suport depenia tant del navegador com del servidor per respondre a la consulta.

<isindex> està desaprovat en HTML 4.01, ja que el mateix comportament es pot aconseguir amb un formulari HTML.

Atributs

Igual que tots els altres elements HTML, aquest element accepta els atributs globals.

prompt
Aquest atribut agrega el seu valor com un indicador de camp de text.
action
Aquest atribut es pot utilitzar per enviar la consulta a un URL diferent. L'acció no es defineix pel W3C.

Exemple

<head>
  <isindex prompt="Search Document... " />
</head>

Historia

Al juny de 1992 Dan Connolly preferiria un tipus d'ancoratge diferent en lloc de isindex.

Al novembre de 1992, els índexs com a enllaços en lloc de documents iniciats per Dan Connolly impulsa la idea que els índexs són més enllaços que documents. En aquest fil, es proposen diferents tipus de solucions. La pregunta dels formularis per fer consultes s'esmenta en referència al navegador de Dynatext: "El navegador mostra botons alternatius, camps de text, etc. L'usuari emplena els camps, fa clic a Acceptar i els resultats de la consulta apareixen en la finestra de la taula de contingut."

En una aproximació sobre isindex al novembre de 1992, Kevin Hoadley va qüestionar la necessitat d'un element isindex i va proposar abandonar-ho. En canvi va proposar tenir un element d'entrada (idea recolzada per Steve Putz). Tim Berners-Lee explica la finalitat resultant de isindex en resultats de cerca agregada. Kevin respon que ell no li agrada la naturalesa booleana de isindex i preferiria un sistema on tot és cerca i proposa estendre el marc actual WWW amb una configuració httpd específica i definir que algunes URI de mapeig crein les consultes de cerca.

En 2016, s'ha proposat per elimnar isindex de l'especificació.

Referència HTML

  • HTML5 la classifica com una característica no conforme.
  • ISINDEX element desaprovat en HTML 4.01
  • ISINDEX en HTML 3.2
  • ISINDEX en HTML 2.0 així com la descripció del comportament en Queries i Indexes (HTML 2.0)
  • ISINDEX en HTML+
