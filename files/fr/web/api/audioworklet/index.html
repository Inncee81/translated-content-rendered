Contexte sécurisé
Cette fonctionnalité est uniquement disponible dans des contextes sécurisés (HTTPS), pour certains navigateurs qui la prennent en charge.

L'interface AudioWorklet dans l'API Web Audio est utilisée pour fournir des scripts de traitement audio personnalisés qui s'exécutent dans un thread séparé afin de fournir un traitement audio à très faible latence. Le code du worklet est exécuté dans le contexte d'exécution global AudioWorkletGlobalScope, en utilisant un thread audio web séparé qui est partagé par le worklet et les autres nœuds audio.

L'accès à distance d'AudioWorklet du contexte audio se fait par la propriété BaseAudioContext.audioWorklet.

Propriétés

L'interface AudioWorklet ne définit pas de propriétés propres, mais hérite des propriétés de Worklet.

Méthodes

Cette interface hérite des méthodes de Worklet. L'interface AudioWorklet ne définit aucune méthode propre.

Evénements

L'AudioWorklet n'a pas d'événements auxquels il répond.

Exemples

Voir AudioWorkletNode pour des exemples complets de création de nœuds audio personnalisés.
