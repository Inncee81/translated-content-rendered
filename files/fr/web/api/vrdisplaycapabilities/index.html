Cette fonction est expérimentale
Puisque cette fonction est toujours en développement dans certains navigateurs, veuillez consulter le tableau de compatibilité pour les préfixes à utiliser selon les navigateurs.
Il convient de noter qu'une fonctionnalité expérimentale peut voir sa syntaxe ou son comportement modifié dans le futur en fonction des évolutions de la spécification.

L'interface VRDisplayCapabilities de WebVR API décrit les capacités d'un appareil VRDisplay — ses fonctionalités peuvent être utilisée pour tester les capacités de l'appareil, par exemple il peut retourner de l'information sur la position.

Cet interface est accessible à partir de la propriété VRDisplay.capabilities.

Properties

VRDisplayCapabilities.canPresent Lecture seule
Retourne un objet Boolean qui décrit si l'écran RV est capable de présenter du contenu (e.g. en utilisant un visiocasque).
VRDisplayCapabilities.hasExternalDisplay Lecture seule
Retourne un objet Boolean indiquant si l'écran RV est séparé de l'écran primaire de l'appareil.
VRDisplayCapabilities.hasOrientation Lecture seule
Retourne un objet Boolean indiquant si l'écran RV peut suivre le mouvement de l'appareil et retourne de l'information sur l'orientation.
VRDisplayCapabilities.hasPosition Lecture seule
Retourne un objet Boolean indiquant si l'écran RV peut suivre le mouvement de l'appareil et retourne de l'information sur l'orientation.
VRDisplayCapabilities.maxLayers Lecture seule
Retourne un nombre indiquant le nombre maximal de VRLayers que l'écran RV peut présenter en même temps (e.g. la longueur maximale du tableau que Display.requestPresent() peut accepter.)
