JavaScript Debugger允许单步调试，进入到你的Javascript代码中审查并修改相应的值，便于追踪缺陷。

你可以运用JS Debugger来调试并运行在远程或本地的代码，比如在火狐OS或安卓设备上。远程调试请查看 远程调试  ，来学习如何将调试器与远程目标连接起来 。

这里有一份快速指南来帮助你熟悉JS Debugger的界面。

如何使用？

你可以参考以下文档来了解如何使用Debugger。

  • 打开Debugger
  • 美化源代码
  • 搜索
  • 使用source map

暂停执行

有时你可能想暂停正在执行的代码，来了解代码的各个部分在做什么。为了告诉Debugger何时暂停执行，有以下多种不同的方法：

  • 设置断点
  • 设置条件断点
  • 设置XHR断点
  • 设置Event Listener断点
  • 设置异常断点
  • 使用watchpoints
  • 设置DOM变化断点
  • 禁用断点

控制执行

代码暂停执行之后能够做什么？

  • 单步执行
  • 忽略源代码
  • 调试worker线程代码
  • 调试eval()中的代码

观察变量

在执行期间或暂停时，有时你可能会想观察变量或表达式的值。

  • 设置日志点
  • 设置watch表达式
